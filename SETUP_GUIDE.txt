FERNVEIL - TILE & FARMING SETUP GUIDE
======================================

FARM LAYER SETUP (one time)
---------------------------
1. Open world.tscn in the Godot editor
2. Select "FarmLayer" in the scene tree
3. In the bottom panel, click the "TileSet" tab
4. Click "+" to add a new atlas source
5. Drag plants.png into the Texture field (this becomes source ID 2)
6. The plants atlas will auto-slice into a 32x32 grid
7. Optionally add plowed_soil.png as another atlas source for a dedicated soil tile

PAINTING CROP PLOTS
--------------------
1. Select "FarmLayer" in the scene tree
2. Switch to the "TileMap" tab in the bottom panel
3. Paint soil tiles wherever you want plantable plots
4. Run the game (F5), walk to a soil tile, press E to plant
5. The tile will cycle through growth stages automatically
6. Press E when fully grown to harvest (tile resets to soil)

DIFFERENT CROP TYPES
---------------------
In FarmLayer's Inspector, change "Crop Column" to pick which crop gets planted:
  Column 0 = generic plant
  Column 1 = herb
  Column 2 = leafy plant
  Column 3 = pepper/tomato

The stage_coords array in scripts/farm_layer.gd maps each column to its
growth stages. Add more entries to support more crop types.

ADDING ROCKS/OBSTACLES (tile painting)
---------------------------------------
1. Select "ObstacleLayer" in the scene tree
2. In TileSet tab, click "+" and add rocks.png (or bushes.png, trees.png)
3. Select rock tiles, expand Physics > Physics Layer 0
4. Draw a collision polygon on each rock tile
5. Switch to TileMap tab, select ObstacleLayer, and paint rocks

ADDING PICKUPS (mushrooms, flowers, items you collect)
-------------------------------------------------------
Pickups use scripts/pickup.gd — press E to collect them, they disappear.
All pickup scenes live in scenes/pickups/.

Placing existing pickups:
1. Right-click "Pickups" node in the scene tree > Instantiate Child Scene (Ctrl+Shift+A)
2. Pick a scene from scenes/pickups/ (e.g. mushroom.tscn)
3. Position it in the viewport
4. Ctrl+D to duplicate and place more
5. Each instance has "Item Name" and "Quantity" properties in the Inspector

Creating a NEW pickup from scratch:
1. Create a new scene (Scene > New Scene)
2. Pick Area2D as the root node
3. Add a Sprite2D child (drag in whatever texture you want)
4. Add a CollisionShape2D child (for interaction detection)
5. Add a StaticBody2D child (for physical collision):
   - Set StaticBody2D's Collision > Layer to 1
   - Add a CollisionShape2D child to it (slightly smaller than the sprite)
6. Select the root Area2D node:
   - Set Collision > Layer to 2 (uncheck layer 1)
   - Set Collision > Mask to none (uncheck all)
   - Drag scripts/pickup.gd into the Script property
7. Set "Item Name" (e.g. "Wildflower"), optionally set "Quantity" and "Message"
8. Save to scenes/pickups/ (e.g. scenes/pickups/wildflower.tscn)
9. Place it in the world under the "Pickups" node (see above)

pickup.gd properties:
  Item Name  - name of the item (used in pickup message, future inventory)
  Quantity   - how many to give on pickup (default 1)
  Message    - custom message (leave blank for auto "Picked up [Item Name]!")

ADDING INTERACTABLE OBJECTS (signs, NPCs, non-collectible things)
------------------------------------------------------------------
Interactables use scripts/interactable.gd — press E to trigger, they stay in the world.
Use these for things like signs, chests, or NPCs that don't disappear.

Placing existing interactables:
1. Right-click the appropriate container node > Instantiate Child Scene (Ctrl+Shift+A)
2. Position it in the viewport
3. Each instance has a "Message" property in the Inspector

Creating a NEW interactable from scratch:
1. Create a new scene (Scene > New Scene)
2. Pick Area2D as the root node
3. Add a Sprite2D child (drag in whatever texture you want)
4. Add a CollisionShape2D child (for interaction detection)
5. Select the root Area2D node:
   - Set Collision > Layer to 2 (uncheck layer 1)
   - Set Collision > Mask to none (uncheck all)
   - Drag scripts/interactable.gd into the Script property
6. In the Inspector, set the "Message" to whatever you want it to say
7. Save the scene (e.g. scenes/sign.tscn, scenes/chest.tscn, etc.)

Adding COLLISION to interactables (so the player can't walk through them):
1. Add a StaticBody2D child node to the root Area2D
2. Set StaticBody2D's Collision > Layer to 1 (physics layer)
3. Add a CollisionShape2D child to the StaticBody2D
4. Give it a shape (CircleShape2D or RectangleShape2D, slightly smaller than the sprite)

For custom behavior beyond printing a message:
  - Duplicate interactable.gd and rename it (e.g. chest.gd)
  - Edit the interact() function with your own logic
  - Attach the new script to your scene instead

DAY/NIGHT CYCLE
-----------------
The DayNightCycle node (CanvasModulate) handles time and screen tinting.
The TimeHUD (CanvasLayer) displays the clock, day number, and phase.

Phases:
  Dawn  (5:00 AM - 7:00 AM)  - warm transition from night to day
  Day   (7:00 AM - 6:00 PM)  - full brightness, normal gameplay
  Dusk  (6:00 PM - 8:00 PM)  - warm transition from day to night
  Night (8:00 PM - 5:00 AM)  - blue-purple tint, supernatural events

Tuning time speed:
  Select DayNightCycle in the scene tree, change "Seconds Per Minute" in Inspector
    1.0 = real-time feel (24 real minutes per full day) — default
    0.1 = fast testing (2.4 real minutes per full day)
    2.0 = slower pace (48 real minutes per full day)

Signals (for hooking other systems):
  phase_changed(new_phase: String)  - fires when dawn/day/dusk/night transitions
  hour_changed(hour: int)           - fires every in-game hour
  new_day(day_number: int)          - fires at midnight

Checking time from other scripts:
  var cycle = get_node("/root/world/DayNightCycle")
  if cycle.is_night():
      # do night-only logic
  var time_text = cycle.get_time_string()  # e.g. "2:30 PM"

REPLACING ART WITH YOUR OWN
-----------------------------
Same grid layout (32x32 tiles):
  - Overwrite the PNG file with your new art, same filename
  - Godot re-imports automatically, all painted tiles update

Different grid/layout:
  - Swap the Texture in the atlas source settings
  - Adjust Texture Region Size if needed
  - Repaint tiles that no longer match

LAYER STRUCTURE
----------------
  GroundLayer  (z_index -1)  - grass, dirt, paths
  DetailLayer  (z_index  0)  - flowers, details
  FarmLayer    (z_index  0)  - plantable soil, growing crops
  ObstacleLayer(z_index  1)  - rocks, trees, walls (with collision)
  Pickups      (Node2D)      - container for collectible items (mushrooms, flowers, etc.)
  Player       (z_index  0)  - CharacterBody2D
  DayNightCycle              - CanvasModulate (screen tint + time tracking)
  TimeHUD      (CanvasLayer) - Clock, day counter, phase display

CONTROLS
---------
  WASD  - Move
  E     - Interact (plant, harvest, talk to objects)

FILE STRUCTURE
---------------
  assets/
    characters/Base.png          - Player spritesheet
    tilesets/
      ground_tileset.png         - Mixel ground tiles (32x32, 12x12 grid)

      bushes.png                 - Mixel bushes
      trees.png                  - Mixel trees
      mushrooms.png              - Mixel mushrooms
      nature_details.png         - Mixel details
      ruins.png                  - Mixel ruins

      plowed_soil.png            - Daneeklu soil
  scenes/
    world.tscn                   - Main world scene
    player.tscn                  - Player scene
    pickups/
      mushroom.tscn              - Collectible mushroom
    crop_plot.tscn               - Standalone crop plot (legacy, use FarmLayer instead)
    rock.tscn                    - Standalone rock obstacle (legacy)
    mushroom.tscn                - Old mushroom (legacy, use pickups/mushroom.tscn instead)
  scripts/
    player.gd                   - Player movement + interaction
    farm_layer.gd               - Tile-based farming system
    pickup.gd                   - Collectible items (disappear on pickup)
    interactable.gd             - Generic interactable (signs, NPCs — stays in world)
    day_night_cycle.gd          - Day/night cycle (time, phases, screen tint)
    time_hud.gd                 - HUD displaying clock, day, and phase
    crop_plot.gd                - Standalone crop logic (legacy, use farm_layer.gd instead)
