FERNVEIL - TILE & FARMING SETUP GUIDE
======================================

FARM LAYER SETUP (one time)
---------------------------
1. Open world.tscn in the Godot editor
2. Select "FarmLayer" in the scene tree
3. In the bottom panel, click the "TileSet" tab
4. Click "+" to add a new atlas source
5. Drag plants.png into the Texture field (this becomes source ID 2)
6. The plants atlas will auto-slice into a 32x32 grid
7. Optionally add plowed_soil.png as another atlas source for a dedicated soil tile

PAINTING CROP PLOTS
--------------------
1. Select "FarmLayer" in the scene tree
2. Switch to the "TileMap" tab in the bottom panel
3. Paint soil tiles wherever you want plantable plots
4. Run the game (F5), walk to a soil tile, press E to plant
5. The tile will cycle through growth stages automatically
6. Prebesets to soil)

SEED SELECTION & PLANTING
--------------------------
Planting requires a seed selected in the inventory:
1. Press Tab to open inventory
2. Click a seed to select it (highlighted green with [SELECTED])
3. Close inventory (Tab), walk to a soil tile, press E to plant
4. Planting consumes 1 seed from inventory
5. Harvesting gives back the crop matching what was planted (e.g. plant Herb → harvest Herb)

You start with 3 of each seed type. The HUD shows your selected seed in the bottom-left.

DIFFERENT CROP TYPES
---------------------
Each crop type is a column in plants.png (32px wide, rows = growth stages).
  Column 0 = Generic Plant
  Column 1 = Herb
  Column 2 = Leafy Plant
  Column 3 = Pepper

REPLACING CROP NAMES & ART WITH YOUR OWN
------------------------------------------
Update these three places to add/rename crops:

1. scripts/inventory.gd — seed_columns dictionary (maps name → sprite column):
     var seed_columns: Dictionary = {
         "Moonpetal": 0,
         "Whispergrass": 1,
     }

2. scripts/inventory.gd — _ready() starter seeds (match names above):
     func _ready() -> void:
         add_item("Moonpetal", 3)
         add_item("Whispergrass", 3)

3. scripts/farm_layer.gd — crop_names array (index = column number):
     var crop_names: Array[String] = [
         "Moonpetal",      # column 0
         "Whispergrass",   # column 1
     ]

To add MORE crop types beyond the existing 4:
1. Add a new column to plants.png (32px wide, 5 rows for growth stages)
2. Add the name→column entry to seed_columns in inventory.gd
3. Add the name to crop_names in farm_layer.gd
4. Add a new row to stage_coords in farm_layer.gd:
     [Vector2i(4, 0), Vector2i(4, 1), Vector2i(4, 2), Vector2i(4, 3), Vector2i(4, 4)],

Everything else (inventory UI, planting, harvesting) picks up the new crops automatically.

ADDING ROCKS/OBSTACLES (tile painting)
---------------------------------------
1. Select "ObstacleLayer" in the scene tree
2. In TileSet tab, click "+" and add rocks.png (or bushes.png, trees.png)
3. Select rock tiles, expand Physics > Physics Layer 0
4. Draw a collision polygon on each rock tile
5. Switch to TileMap tab, select ObstacleLayer, and paint rocks

ADDING PICKUPS (mushrooms, flowers, items you collect)
-------------------------------------------------------
Pickups use scripts/pickup.gd — press E to collect them, they disappear.
All pickup scenes live in scenes/pickups/.

Placing existing pickups:
1. Right-click "Pickups" node in the scene tree > Instantiate Child Scene (Ctrl+Shift+A)
2. Pick a scene from scenes/pickups/ (e.g. mushroom.tscn)
3. Position it in the viewport
4. Ctrl+D to duplicate and place more
5. Each instance has "Item Name" and "Quantity" properties in the Inspector

Creating a NEW pickup from scratch:
1. Create a new scene (Scene > New Scene)
2. Pick Area2D as the root node
3. Add a Sprite2D child (drag in whatever texture you want)
4. Add a CollisionShape2D child (for interaction detection)
5. Add a StaticBody2D child (for physical collision):
   - Set StaticBody2D's Collision > Layer to 1
   - Add a CollisionShape2D child to it (slightly smaller than the sprite)
6. Select the root Area2D node:
   - Set Collision > Layer to 2 (uncheck layer 1)
   - Set Collision > Mask to none (uncheck all)
   - Drag scripts/pickup.gd into the Script property
7. Set "Item Name" (e.g. "Wildflower"), optionally set "Quantity" and "Message"
8. Save to scenes/pickups/ (e.g. scenes/pickups/wildflower.tscn)
9. Place it in the world under the "Pickups" node (see above)

pickup.gd properties:
  Item Name  - name of the item (used in pickup message, future inventory)
  Quantity   - how many to give on pickup (default 1)
  Message    - custom message (leave blank for auto "Picked up [Item Name]!")

ADDING INTERACTABLE OBJECTS (signs, NPCs, non-collectible things)
------------------------------------------------------------------
Interactables use scripts/interactable.gd — press E to trigger, they stay in the world.
Use these for things like signs, chests, or NPCs that don't disappear.

Placing existing interactables:
1. Right-click the appropriate container node > Instantiate Child Scene (Ctrl+Shift+A)
2. Position it in the viewport
3. Each instance has a "Message" property in the Inspector

Creating a NEW interactable from scratch:
1. Create a new scene (Scene > New Scene)
2. Pick Area2D as the root node
3. Add a Sprite2D child (drag in whatever texture you want)
4. Add a CollisionShape2D child (for interaction detection)
5. Select the root Area2D node:
   - Set Collision > Layer to 2 (uncheck layer 1)
   - Set Collision > Mask to none (uncheck all)
   - Drag scripts/interactable.gd into the Script property
6. In the Inspector, set the "Message" to whatever you want it to say
7. Save the scene (e.g. scenes/sign.tscn, scenes/chest.tscn, etc.)

Adding COLLISION to interactables (so the player can't walk through them):
1. Add a StaticBody2D child node to the root Area2D
2. Set StaticBody2D's Collision > Layer to 1 (physics layer)
3. Add a CollisionShape2D child to the StaticBody2D
4. Give it a shape (CircleShape2D or RectangleShape2D, slightly smaller than the sprite)

For custom behavior beyond printing a message:
  - Duplicate interactable.gd and rename it (e.g. chest.gd)
  - Edit the interact() function with your own logic
  - Attach the new script to your scene instead

DAY/NIGHT CYCLE
-----------------
The DayNightCycle node (CanvasModulate) handles time and screen tinting.
The TimeHUD (CanvasLayer) displays the clock, day number, and phase.

Phases:
  Dawn  (5:00 AM - 7:00 AM)  - warm transition from night to day
  Day   (7:00 AM - 6:00 PM)  - full brightness, normal gameplay
  Dusk  (6:00 PM - 8:00 PM)  - warm transition from day to night
  Night (8:00 PM - 5:00 AM)  - blue-purple tint, supernatural events

Tuning time speed:
  Select DayNightCycle in the scene tree, change "Seconds Per Minute" in Inspector
    1.0 = real-time feel (24 real minutes per full day) — default
    0.1 = fast testing (2.4 real minutes per full day)
    2.0 = slower pace (48 real minutes per full day)

Signals (for hooking other systems):
  phase_changed(new_phase: String)  - fires when dawn/day/dusk/night transitions
  hour_changed(hour: int)           - fires every in-game hour
  new_day(day_number: int)          - fires at midnight

Checking time from other scripts:
  var cycle = get_node("/root/world/DayNightCycle")
  if cycle.is_night():
      # do night-only logic
  var time_text = cycle.get_time_string()  # e.g. "2:30 PM"

DIALOGUE SYSTEM
----------------
Walk up to an NPC and press E to start dialogue.
Press E to advance text, or click a choice button when choices appear.
Text displays with a typewriter effect — press E to skip to full text.

Placing an NPC:
1. Instance scenes/npc.tscn into world.tscn (under NPCs node)
2. In Inspector, set:
   - "Npc Name" — display name in dialogue box
   - "Dialogue Id" — which dialogue to trigger (key in dialogue_manager.gd)
   - "Sprite Color" — placeholder color (ignored when real art is added)
3. Position in the viewport

Replacing NPC placeholder with real art:
  Add a Sprite2D or AnimatedSprite2D child — the script skips the placeholder.

  Using a static sprite (Sprite2D):
  1. Select the NPC instance in the scene tree (e.g. NPCs > TestNPC)
  2. Add a Sprite2D child node
  3. Set Texture to your character image
  4. If using a spritesheet, enable Region > Enabled = true
  5. Set Region Rect to a single frame (e.g. x:0, y:0, w:48, h:48)
  6. Optionally set Modulate to a different color to distinguish from the player

  Using the player spritesheet as a quick test:
  1. Add Sprite2D child, set Texture to assets/characters/Base.png
  2. Enable Region, set Rect to one frame
  3. Tint with Modulate color in Inspector

  For animated NPCs:
  Use AnimatedSprite2D instead, set up idle animations like the player.

Adding new dialogue:
  Edit scripts/dialogue_manager.gd, add entries to the dialogues dictionary.
  Each dialogue is an Array of lines:
    {"speaker": "Name", "text": "Hello!"}
  Add choices to branch:
    {"speaker": "Name", "text": "Question?", "choices": [
        {"text": "Option A", "next": "other_dialogue_id"},
        {"text": "Option B", "next": "another_id"},
    ]}

Signals (for hooking other systems):
  dialogue_started()    - fires when dialogue begins
  dialogue_ended()      - fires when dialogue finishes
  dialogue_line_shown(speaker, text) - fires each line

CRAFTING SYSTEM
----------------
5 crafting stations: Drying Rack, Mortar & Pestle, Kettle, Press, Bouquet Table.
Walk up to a station and press E to open its crafting menu.

Recipes show ingredient counts (green = have enough, red = not enough).
Click "Craft" to consume ingredients and produce the result.
Press E or Esc to close the crafting UI.

Placing a crafting station in the world:
1. Create a new scene: Area2D root
2. Add a Sprite2D child (station art)
3. Add a CollisionShape2D child (for interaction, layer 2)
4. Add a StaticBody2D child with its own CollisionShape2D (for collision, layer 1)
5. Set the root Area2D:
   - Collision > Layer = 2, Mask = none
   - Script = scripts/crafting_station.gd
6. In Inspector, set "Station Type" to one of:
   Drying Rack, Mortar & Pestle, Kettle, Press, Bouquet Table
7. Save to scenes/ (e.g. scenes/drying_rack.tscn)
8. Instance it in world.tscn

Adding/changing recipes:
  Edit scripts/crafting_recipes.gd — each station has an array of recipes.
  Each recipe has: ingredients (dict), result (string), result_count (int).

Recipe chain example:
  Herb (harvest) → Dried Herb (Drying Rack) → Herb Powder (Mortar & Pestle) → Plant Tonic (Kettle)

PAUSE MENU & SAVE/LOAD
------------------------
Press Esc to open the pause menu. It has:
  - Resume — close menu and unpause
  - Save Game — save current state to disk
  - Load Game — load the last save
  - Quit — exit the game

Saves to user://savegame.json (on Windows: %APPDATA%\Godot\app_userdata\Fernveil\).

What gets saved:
  - Inventory (all items + selected seed)
  - Day/night cycle (hour, minute, day number)
  - Player position
  - Farm crops (growth stages, timers)

You can also call from code:
  SaveManager.save_game()   # returns true on success
  SaveManager.load_game()   # returns true on success
  SaveManager.has_save()    # check if a save file exists
  SaveManager.delete_save() # delete the save file

To add new data to save/load:
  1. Open scripts/save_manager.gd
  2. In save_game(), gather state into the data dictionary
  3. In load_game(), restore state from the loaded data
  Note: Dictionary keys must be strings for JSON. Use "x,y" format for Vector2i keys.

Customizing the pause menu:
  Edit scripts/pause_menu.gd — add new buttons in the _build_ui() function.
  The "Options" button is a good future addition for audio/display settings.

REPLACING ART WITH YOUR OWN
-----------------------------
Same grid layout (32x32 tiles):
  - Overwrite the PNG file with your new art, same filename
  - Godot re-imports automatically, all painted tiles update

Different grid/layout:
  - Swap the Texture in the atlas source settings
  - Adjust Texture Region Size if needed
  - Repaint tiles that no longer match

LAYER STRUCTURE
----------------
  GroundLayer  (z_index -1)  - grass, dirt, paths
  DetailLayer  (z_index  0)  - flowers, details
  FarmLayer    (z_index  0)  - plantable soil, growing crops
  ObstacleLayer(z_index  1)  - rocks, trees, walls (with collision)
  Pickups      (Node2D)      - container for collectible items (mushrooms, flowers, etc.)
  Player       (z_index  0)  - CharacterBody2D
  DayNightCycle              - CanvasModulate (screen tint + time tracking)
  TimeHUD      (CanvasLayer) - Clock, day counter, phase display
  InventoryUI  (CanvasLayer) - Inventory panel (Tab to toggle)
  CraftingUI   (CanvasLayer) - Crafting panel (opens via station interaction)
  NPCs         (Node2D)      - container for NPC characters
  DialogueUI   (CanvasLayer) - Dialogue box (bottom of screen)
  PauseMenu    (CanvasLayer) - Pause menu (Esc to toggle)

CONTROLS
---------
  WASD  - Move
  E     - Interact (plant, harvest, talk to objects)
  Q     - Cycle selected crop type (Generic Plant → Herb → Leafy → Pepper)
  Tab   - Open/close inventory
  Esc   - Pause menu (save, load, quit)
